Download VistA

VistA is designed in a client-server architecture. VistA's server is written in MUMPS, and must be executed in a MUMPS supporting object oriented database engine. Currently, there are only two: Intersystems Caché and FIS GT.M. This source code must be properly configured and installed, and is recommended only for advanced users. For more information on MUMPS, and basic instructions on how to build a development instance, visit the resources section of this site.

Download CPRS Client

While VistA's command line or 'roll-and-scroll' interface is usable via the shell prompt or remote sessions, clients which connect to server instances provide a more intuitive interface to the average user. VistA has several interfaces available, the most robust of which is CPRS (Computerized Patient Record System) and the related suite of Delphi-based clients; these are currently in use within the Department of Veterans Affairs today.

Source Code

OSEHRA VistA uses Git for Source Code Management.

The VistA repositories are mirrored across three sites, osehra.org, github.com, and gitorious.com.

VistA is divided between two primary repositories, VistA-M and VistA. The VistA-M repository is a static representation of VistA as it would appear on a file system at a point in time. This code is not intrinsically functional and must be imported and maintained in a supporting object oriented database engine to run. The VistA repository provides a multitude of supporting utilities and patches required to instantiate and maintain a VistA-M instance.

To contribute to VistA, a local copy of the Git repository should be created for changes to be committed to. To do so, git should be installed on the development machine. Instructions for installing and configuring Git can be found at git-scm.com

VistA-M Repository: 

git clone git://code.osehra.org/VistA-M.git

VistA Repository: 

git clone git://code.osehra.org/VistA.git

CPRS Source Code is currently maintained in the VistA repository. It requires Delphi 2007 to compile, and is found in the repository at the following location

VistA/Packages/Order Entry Results Reporting/CPRS

For more information on how to submit changes for inclusion in these repositories, please visit the Core Development Section of the Community page.

Incremental Patches

Running VistA instances are updated via a series of interrellated patches. These patches take the form of KIDS (Kernel Improvement Distribution System) files. For a better understanding of how to apply patches, review the Patching section of the Resources page. An in-depth discussion of current and future state patching activities is ongoing within the community, and can best be understood by exploring either the OSEHRA Discussions or the Hardhats Mailing List.

Patches are currently maintained by OSEHRA in the VistA Git repository, and can be found using the naming convention below by substituting in the appropriate package name.

VistA/Packages/[Package Name]/Patches
